I. Overview
The SOCKS5 protocol provides one-way communication mechanisms via the CONNECT command. This way a client can send a request
via the SOCKS5 server to another remote server.

Currently only the NO_AUTH option is supported, but further auth options must be implemented in the socks5/auth package.
The proxy its self is implemented in the socks5/proxy package. The proxy its self is simple splice between two io.ReadWriters.

The packages socks5/requests and socks5/responses provide utilities for encapsulating data into the SOCKS5 formats.

The socks5/shared package contains data structures used by both the request and the responses.
